1.默认排序
	按照查询和文档的相关度排序。默认按评分降序排序
	对于match_all query而言 返回所有文档 不需要评分 文档的顺序即为添加文档的顺序。如果要对其排序可以用_doc降序。
	GET /_search {
		"query": {...},
		"sort": [
			{"_score": {"order": "desc"}}
		]
	}

2.多字段排序
	GET /_search {
		"query": {...},
		"sort": [
			{"price": {"order": "desc"}},
			{"year": {"order": "asc"}}
		]
	}


3.分片影响评分
	es5.4之后对于text类型的字段使用BM25评分模型 而不是基于tf-idf的向量空间模型。
	评分模型的选择可以在similarity参数在映射中指定。
	es在每个分片上单独打分 分片的数量会影响打分的结果。
	分词器也会影响评分 因为使用不同的分词器会使倒排索引中的词项数发生改变
